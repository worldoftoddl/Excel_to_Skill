# DEVIATIONS — 편차 기록 (작업지시서 §10)

지시서와 실물이 충돌하면 실물이 이긴다 — 단, 기록한다.
형식: `D-번호 / 지시서 조항 / 실물 증거 / 취한 조치`.

## D-01 — used range 재계산 (승인, 2026-07-08)

- **지시서 조항**: v1.0 §4.1 `meta.sheets[].dimensions` — 산출 기준 미명시.
- **실물 증거**: 파일의 dimension 레코드 과대 기록 — 감사계약 `1200` 시트,
  레코드 `A2:H43` vs 36행 이하 실콘텐츠 0건.
- **취한 조치**: dimension 레코드를 신뢰하지 않고 **콘텐츠 실재 기준**
  (값·수식이 있거나 §4.4 포함 규칙상 유의 서식 — 테두리·배경색 — 이 있는 셀)의
  최대 행·열로 재계산. 원점은 A1 고정. 오너 재검증으로 승인되어
  v1.1 §5에 규칙으로 명문화됨(v1.1 부록 B에서 본 파일로 전기).

## D-02 — 시트 dependency의 의미 분류 미구현 (관찰, 2026-07-12)

- **대상 실물**: `감사조서서식_4000 계정별 실증절차 (KIFRS용) 2025.xlsx`.
- **실물 증거**: `C` 시트가 `A` 시트를 직접 참조하는 수식은 네 건뿐이다.
  `C!C4 -> A!C4`(회사명), `C!G4 -> A!G4`(작성자),
  `C!C5 -> A!C5`(결산일), `C!G5 -> A!G5`(검토자)이며 현재 값은 모두
  비어 있다. `A` 시트의 현금및현금성자산·장단기금융상품 실증절차 본문은
  `C` 시트에서 참조하지 않는다.
- **영향**: 현재 `audit-scopes`의 `dependency_sheets: ["A"]`는 직접 수식
  의존성을 정확히 표시하지만, 공통 머리말 승계와 실질 감사자료 의존성을 구분하지
  않는다. 이 값을 그대로 후속 분석 대상으로 해석하면 불필요한 `A` 시트 분석을
  권고할 수 있다. 반대로 이번 사례에서 `C` 단독 분석으로 누락된 실질 감사증거는 없다.
- **현재 조치**: dependency 시트명은 수식 참조 표시일 뿐 그 시트의 셀 내용을
  관찰하거나 증거로 사용한 것이 아니라는 경계를 brief·agent 모델 입력에 유지한다.
- **후속 개선**: 참조 대상의 역할을 `header_metadata`, `substantive_data`,
  `calculation_dependency`로 분류한다. 추가 시트 분석 권고에는
  `substantive_data`와 `calculation_dependency`를 우선 사용하고,
  `header_metadata`는 값이 비어 있지 않을 때만 범위 메타데이터로 보강한다.
